1# EXPLICIT CONTROL:
2
3Enable explicit processing with curly brackets.
4For inline processing use {a} without new lines and
5for block processing use {
6 a
7} with surrounding new lines (and optional indentation).
8Anything inside curly brackets has to be a valid expression.
9
10Enable explicit inline latex math with \(a\) and
11explicit block latex math with \[a.\] No
12processing is performed and nothing is enforced for explicit latex math.
13
14No processing is performed inside inline markdown formatting
15such as *italic*, **bold**, ***bolditalic***, ~~strikethrough~~, `verbatim` or
16[links](http://github.com). Processing is performed inside "block" markdown formatting such as headers, lists or quotes.
17
18Disable implicit processing with md-code fences.

20# IMPLICIT PROCESSING:
21
22Anything for which processing begins with an indented new line will be
23block processed, otherwise it will be inline processed.
24
25Any valid binary mathematical operation will enter processing mode
26except for `-`, `@` and word operators such as `in`.
27
28For example, will be processed, but non-positive or foo@bar.com won't be.
29
30Calls such as {f(x)}, {f[x]} or {f{x}} work.
31
32Tuples: {(a,)}, {(a,b,)}, {(a,b,c)}
